# packages/py-shared

The `gxp-shared` Python library. Every FastAPI microservice depends on this package for authentication, tenant context resolution, audit event emission, cross-tenant access control, Valkey Streams messaging, and OpenTelemetry instrumentation. It is a `uv` workspace member — changes here are immediately available to all services without re-installing.

---

## Key Design Decisions

- **JWT validation is fully offline** — `jwt_validator.decode_token` derives the JWKS endpoint from the token's own `iss` claim (`{keycloak_url}/realms/{realm}/protocol/openid-connect/certs`), caches per-realm `PyJWKClient` instances for one hour, and never calls Keycloak at request time. This is critical for air-gap resilience.
- **Multi-realm by design** — because the JWKS URL is derived from the token itself, new tenant realms (`gxp-{slug}`) are handled automatically without any service restart or configuration change.
- **`TenantContextMiddleware` is the single chokepoint** — it validates the JWT, extracts the tenant slug from the realm name in `iss`, confirms the tenant is `active` in Valkey, and writes `request.state.{tenant_slug, user_id, user_email, user_roles}`. The `get_db` dependency (generated by `make_tenant_db_dependency`) then sets `search_path` to `t_{slug}` before yielding the SQLAlchemy session.
- **Audit via Redis Streams (fire-and-forget)** — `emit_audit_event` writes to the `audit:events` Valkey stream. If the audit-service is temporarily down, events persist on disk in Valkey and are consumed when it recovers (NIST 800-53 AU-5).
- **Cross-tenant grants use a short-lived Valkey cache** — `assert_cross_tenant_grant` checks a 60-second cache before hitting the `platform.cross_tenant_grants` table. Revocations propagate within the TTL.

## Compliance / Security Notes

- **AU-5 (NIST 800-53)** — `emit_audit_event` writes to Valkey Streams which persist to disk. Events are not lost if the audit-service is down.
- **AC-3 / IA-5** — `require_roles` and `require_platform_admin` enforce access control at the FastAPI layer, independent of Traefik-level auth.
- JWKS cache TTL is 3600 s. A compromised key would be rotated in Keycloak; services will pick up the new key on the next JWKS refresh.
- `decode_token_unverified` is provided for internal use only (e.g., after Traefik ForwardAuth already verified the token). It must never be used as the primary auth check.
